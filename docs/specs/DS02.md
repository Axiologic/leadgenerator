# DS02: Recursive Scraper Module

## Overview
The Scraper tab crawls a website recursively, extracts text from each page (HTML, PDF, DOCX), and uses an LLM to identify specific leads from the content.

## User Flow
1. User enters a starting URL and depth (1-5)
2. Clicks "Start Scraper"
3. Server processes up to 10 pages per batch
4. Results appear as review cards (same as Discovery)
5. If more pages remain, "Continue тЖТ" button appears
6. User can keep clicking Continue to process more batches
7. Each lead can be individually Rejected or Added to Leads

## Pagination
- **Batch size**: 10 pages per request
- **State**: Server returns `{ leads, visited, knownNames, hasMore, queue }`
- **Continue**: Frontend sends back `visited`, `queue`, and `knownNames` to resume
- **Status**: Shows "X pages visited тАФ more pages available" or "тАФ done"

## URL History
- Stored in `localStorage` (key: `scraperHistory`, max 15 entries)
- Displayed as chips showing hostname (full URL in tooltip)
- Click to fill the URL input

## Page Fetching & Caching
- **Cache location**: `data/cache/` directory
- **Cache key**: MD5 hash of URL тЖТ filename `{hash}.{ext}`
- **Extensions**: `.html` for web pages, `.pdf` for PDFs, `.doc`/`.docx` for Word
- **Cache hit**: Read from disk, skip HTTP request
- **Cache miss**: Fetch via HTTP, save to disk, then process

## Document Text Extraction
See DS03 for PDF and DOCX extraction details.

## Content Cleaning (HTML)
Before sending to LLM, HTML is cleaned by `extractMainContent()`:
1. Remove `<script>`, `<style>`, `<nav>`, `<footer>`, `<header>`, `<aside>`, `<noscript>` blocks
2. Remove HTML comments
3. Strip all remaining HTML tags
4. Collapse whitespace
5. Truncate to 8000 characters

**Rationale**: Without this, LLM extracts boilerplate entities (site name, "European Commission" from footer) instead of actual page-specific leads.

## Deduplication
- Leads are deduplicated by name (case-insensitive) within a batch
- `knownNames` set persists across Continue batches
- Prevents the same entity from appearing multiple times across pages

## LLM Extraction Prompt
The prompt specifically asks for:
- "SPECIFIC people and organizations" тАФ not the website owner
- "project coordinators, researchers, partner companies, contact persons"
- Explicitly says "Do NOT include generic entities like European Commission unless they are a specific contact"

## Link Extraction
- Only follows links within the same hostname (no external crawling)
- Deduplicates URLs before adding to queue
- Respects depth limit per link level

## Error Handling
- Per-page fetch errors: logged, page skipped, scraping continues
- Per-page LLM extraction errors: logged, returns empty array, continues
- Overall scrape errors: returned as 502 to frontend
- Pages with < 50 chars of extracted text are skipped (no LLM call)
