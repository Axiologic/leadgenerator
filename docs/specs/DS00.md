# DS00: Lead Generator — System Overview

## Vision
A lightweight, zero-dependency web application for discovering and managing marketing leads. Uses LLMs (via AchillesAgentLib) and web scraping to find people and organizations, extract contact information, and provide a management interface.

## Architecture

```
┌─────────────────────────────────────────────────┐
│  Browser (SPA)                                  │
│  index.html + app.js + style.css                │
│  Tabs: Discovery | Scraper | Leads | Settings   │
│  localStorage: searchHistory, scraperHistory     │
└──────────────────┬──────────────────────────────┘
                   │ HTTP JSON API
┌──────────────────┴──────────────────────────────┐
│  server.mjs (Node.js http, no dependencies)     │
│  Routes: /api/leads, /api/config, /api/models,  │
│          /api/discovery, /api/suggest-topics,    │
│          /api/scrape, /api/test-model, /api/login│
└──────┬───────────┬──────────────────────────────┘
       │           │
┌──────┴──┐  ┌─────┴──────────────────────────────┐
│ storage │  │ marketingAgent.mjs                  │
│  .mjs   │  │  ├─ discoverLeads(topic)            │
│         │  │  ├─ suggestTopics(topic, history)    │
│ data/   │  │  ├─ recursiveScrape(url, depth, ...) │
│ leads   │  │  └─ extractLeads(content, url)       │
│ config  │  │                                      │
└─────────┘  │  Uses: LLMAgent from AchillesAgentLib│
             │  Uses: pageCache.mjs for disk cache  │
             └──────────────────────────────────────┘
```

## Data Model
- `data/leads.json` — `[{ id, name, type, organization, email, linkedin, website, contactUrl, source, status, notes, score, createdAt }]`
- `data/config.json` — `{ passwordHash, scoring: { keywords }, tasks: { discovery, suggest, extraction, scoring } }`
- `data/cache/` — One file per URL (MD5 hash), HTML/PDF/DOCX stored as-is

## LLM Task Configuration
Each task can be configured independently with a specific model or tier:
| Task | Purpose | Default |
|------|---------|---------|
| `discovery` | Find leads for a topic | tier: fast |
| `suggest` | Generate search query suggestions | tier: fast |
| `extraction` | Extract leads from scraped pages | tier: fast |
| `scoring` | Score/prioritize leads | tier: fast |

**Important:** Search models (exa-search, Tavily-search, etc.) should NOT be used for discovery/suggest/extraction — they return search results, not structured JSON. The Settings UI marks them as `(search)` to avoid confusion.

## API Endpoints
| Method | Path | Description |
|--------|------|-------------|
| POST | /api/login | Authenticate with password |
| GET | /api/leads | List all leads |
| POST | /api/leads | Create a lead |
| PATCH | /api/leads/:id | Update lead (status, notes) |
| DELETE | /api/leads/:id | Delete a lead |
| GET | /api/config | Get config (excludes passwordHash) |
| POST | /api/config | Merge-update config |
| GET | /api/models | List available LLM models |
| POST | /api/test-model | Test a model/tier responds |
| POST | /api/discovery | Discover leads for a topic |
| POST | /api/suggest-topics | Get search suggestions |
| POST | /api/scrape | Scrape URL with pagination |

## Error Handling
- LLM errors (502, bad gateway, quota exceeded) propagate to frontend as `{ error: "message" }` with HTTP 502
- Frontend shows errors inline with red `❌` prefix
- LLM cascade in AchillesAgentLib skips models without API keys (no wasted calls)
- Settings auto-save with toast notification (no blocking alerts)
