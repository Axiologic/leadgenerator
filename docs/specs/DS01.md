# DS01: Discovery Module

## Overview
The Discovery tab allows users to find leads (people and organizations) by entering a search topic. An LLM identifies relevant entities and classifies them.

## User Flow
1. User enters a topic in the search input (or clicks a suggestion/history item)
2. Clicks "Search" (or presses Enter)
3. LLM returns 5-10 classified results
4. Each result is shown as a card with type badge, name, and contact links
5. User can Reject (dismiss) or Add to Leads (save) each result

## Search Suggestions
- "Suggest" button generates 5 contextual search queries via LLM
- Uses the `suggest` task (separately configurable model/tier)
- If a topic is entered, suggestions are related to it
- If search history exists, LLM is told to suggest different queries
- Suggestions appear as clickable chips above results

## Search History
- Stored in `localStorage` (key: `searchHistory`, max 20 entries)
- Displayed as chips labeled "Recent:" below the search input
- Click a chip to fill the input (does not auto-search)
- Most recent searches appear first

## Lead Classification
The LLM prompt requires each result to have:
- `type`: `"person"` or `"organization"`
- For persons: `name`, `email`, `linkedin`
- For organizations: `name`, `website`, `contactUrl`

## LLM Response Parsing
See DS03 for the robust JSON parsing strategy used across all LLM interactions.

## Card UI
- **Badge**: Blue "PERSON" or green "ORGANIZATION"
- **Links**: LinkedIn/email for persons, Website/Contact for organizations
- **Reject**: Removes card from DOM
- **Add to Leads**: POST to /api/leads, card greys out with "✅ Added"
- **Error**: If add fails, shows inline red error on the card

## Error States
- Empty topic → search button does nothing
- LLM failure → red error message in results area with full error text
- No results → "No results found." message
