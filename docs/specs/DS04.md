# DS04: Lead Management & Settings

## Lead Management

### Lead List View
- Leads displayed as clickable rows (not a table — responsive flex layout)
- Each row shows: type badge, name, primary contact link, status badge, source
- **Filters**: Status dropdown (All/New/Contacted/Not Interested) + Type dropdown (All/Person/Organization)
- **Counter**: "X of Y leads" showing filtered vs total count
- Empty state: "No leads yet. Use Discovery or Scraper to find some."

### Lead Detail Modal
Opens on clicking any lead row. Contains:
- All lead fields displayed as label: value pairs
- Links are clickable (mailto: for email, target=_blank for URLs)
- **Status dropdown**: New / Contacted / Not Interested
- **Notes textarea**: Free-form text, persisted on save
- **Save Changes**: PATCH /api/leads/:id with status + notes
- **Delete Lead**: DELETE /api/leads/:id with confirmation dialog
- Click outside modal or × button to close

### Status Workflow
```
New → Contacted → (keep notes about interaction)
New → Not Interested → (dismissed)
Contacted → Not Interested → (didn't work out)
```
Status badges are color-coded: blue (New), green (Contacted), red (Not Interested).

## Settings

### LLM Task Configuration
Four independently configurable tasks:
- **Discovery**: Model used for `POST /api/discovery`
- **Suggest**: Model used for `POST /api/suggest-topics`
- **Extraction**: Model used for scraper lead extraction
- **Scoring**: Model used for lead scoring (future)

Each task has:
- Dropdown with all available models grouped by: tiers (fast/deep), fast models, deep models, search models
- "Test" button that sends a simple prompt and checks for "OK" response
- Status indicator (⏳/✅/❌)

**Model filtering in UI:**
- Search models (exa-search, Tavily, DuckDuckGo) listed separately as `(search)` 
- If a saved model no longer exists in the model list, it's shown as `(saved)` to prevent silent reset

### Auto-Save Behavior
- No "Save" button — all changes save immediately
- Dropdown change → auto-save + toast "✓ discovery → copilot-gpt-4.1"
- Keyword add → auto-save + toast "✓ Added "innovation" (10 pts)"
- Keyword remove → auto-save + toast "✓ Removed "innovation""
- Toast appears bottom-right, green, fades after 2 seconds

### Scoring Keywords
- Display current keywords with point values
- Add: text input + number input + "Add" button
- Remove: × button per keyword
- Keywords stored in `config.scoring.keywords` as `{ keyword: points }`
- Used for future automated lead prioritization

## Authentication
- Simple password-based (passwordHash in config, null = no auth)
- Token = password itself (stored in localStorage)
- 401 response triggers password prompt
- All API calls except /api/login require valid token when password is set
